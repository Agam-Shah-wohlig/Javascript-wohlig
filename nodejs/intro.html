Node.js Notes

Summary

Node.js enables JavaScript to run outside the browser by embedding Chrome’s V8 engine into C++. This gives JavaScript system-level capabilities and makes it possible to create servers, work with files, and build backend applications. Node.js is a runtime environment, not a framework or library. It was created by Ryan Dahl in 2009.

Detailed Notes

Creator of Node.js

  • Ryan Dahl created Node.js in 2009.

What is JavaScript?

  • JavaScript is a browser language used for interactivity.
  • It executes inside browsers because they include JS engines.

JavaScript Engines

  • Chrome: V8 Engine
  • Firefox: SpiderMonkey
  • Safari: JavaScriptCore
  • V8 is the most popular engine.

Before Node.js

  • JavaScript could not run outside browsers.
  • No way to execute JS in terminal or interact with system resources.

Creation of Node.js

  • Chrome’s V8 engine was extracted.
  • It was embedded into C++ for machine-level capabilities.
  • This allowed JavaScript to run outside browsers.
  • Created by Ryan Dahl.

What is Node.js?

  • Node.js is NOT a framework.
  • Node.js is NOT a library.
  • Node.js is a Runtime Environment for JavaScript.

What Node.js Enables

  • Run JavaScript outside the browser
  • Create servers
  • Build REST APIs
  • Perform file handling
  • Access machine-level features

Node REPL

  • Using node in terminal opens an interactive JS environment.
  • You can run code like console.log() or math operations.

Extra Notes

  • Official website: nodejs.org
  • Node.js is open-source and cross-platform.
  • Basic JavaScript knowledge is recommended before learning Node.js.
