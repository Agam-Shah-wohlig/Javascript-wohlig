HTTP Headers - Notes

1. What are HTTP Headers?

    HTTP headers are additional information sent along with HTTP requests and responses.
  • Request and response packets contain actual data in the body.
  • Headers provide metadata about the request/response like origin, content type, length, etc.
  • Headers allow the server and client to understand and manage the data correctly.

2. Analogy

  • Mail analogy: Sending a letter with extra details (from, to, weight) is like an HTTP request with headers.
  • The actual message inside the envelope is the body of the HTTP request/response.

3. Definition

  • HTTP Headers carry metadata about request and response bodies.
  • They include extra information such as data type, encoding, connection info, etc.

4. Examples in the Real World

Example 1: YouTube Network Tab
- Open YouTube in Incognito mode and inspect Network tab.
- Request headers include: Method, Path, Scheme, Accept, Accept-Language, Cache-Control, Cookies, User-Agent.
- Response headers include: X-Powered-By, Content-Type, Content-Length, Connection.
- Headers carry info such as which content type is expected, language preference, etc.
Example 2: Postman Request
- Sent GET request to localhost:8000/api/users.
- Request headers automatically included by Postman: User-Agent, Accept, Postman-Token, Host, Accept-Encoding, Connection.
- Response headers include: X-Powered-By (Express), Content-Type, Content-Length, Connection.
Example 3: Custom Headers
- You can add custom headers using response.setHeader().
- Best practice: Prefix custom headers with X-.
- Example: X-My-Name: Piyush Garg added to response headers.
- Helps in identifying custom metadata without conflicting with built-in headers.
Example 4: Postman Content-Type Handling
- While sending POST requests, Postman sets Content-Type automatically:
  • For x-www-form-urlencoded form, header is application/x-www-form-urlencoded.
  • For JSON body, header is application/json.
- Middleware on server reads these headers to parse the request body correctly.

5. Best Practices

  • Always prefix custom headers with X- (e.g., X-Custom-Header).
  • Use headers to provide extra info without modifying the actual body data.
  • Headers are useful for authentication, content negotiation, caching, and client/server metadata.

6. Summary

  • HTTP headers = metadata for request/response.
  • Headers include built-in headers (Content-Type, User-Agent) and custom headers (X-My-Header).
  • Headers are inspected using browser dev tools (Network tab) or tools like Postman.
  • Best practice: Use X- prefix for custom headers.
