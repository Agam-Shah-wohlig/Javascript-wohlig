Versioning Notes (Summary)

1. Version Structure (Semantic Versioning)

A version is written as MAJOR.MINOR.PATCH, e.g., 4.18.2.

  • PATCH (2) – Minor bug fixes, optional, safe to update.
  • MINOR (18) – Recommended updates, security fixes, new features.
  • MAJOR (4) – Breaking changes, may break existing code. Update carefully.

2. Meaning of Updates

  • Patch: Very small, safe fixes.
  • Minor: Recommended, includes improvements & security fixes.
  • Major: Big changes, may break project (e.g., Express 4 → 5).

3. Installing Packages

  • Latest version: npm install express
  • Specific version: npm install express@4.17.2
  • Specific major version: npm install express@4

4. Version Symbols

^ Caret

"express": "^4.18.2"

  • Locks MAJOR version (4)
  • Updates MINOR + PATCH automatically
  • Will NOT go to 5.x

~ Tilde

"express": "~4.18.2"

  • Updates only PATCH version
  • Does NOT update MINOR

No Symbol (Exact version)

"express": "4.18.2"

  • Fully locked
  • No automatic updates

Range Examples

  • >=4.0.0 <5.0.0
  • 4.18.x

5. Security & Stability Risks

  • A wrong version string can break your entire project.
  • Never use "latest" — unstable & unpredictable.
  • Always check changelogs before updating versions.

6. Best Practices

  • Use ^ for general development.
  • Use ~ for stable production apps.
  • Lock major versions.
  • Review changelog before updating MAJOR or MINOR.
