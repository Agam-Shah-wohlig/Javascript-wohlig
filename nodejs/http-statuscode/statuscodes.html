HTTP Status Codes - Notes

1. What are HTTP Status Codes?

  • Status codes are standard codes sent in HTTP responses to indicate the result of a request.
  • They help the client understand whether the request succeeded, failed, or needs further action.
  • Each response has a code that belongs to one of five categories.

2. Categories of Status Codes

  • 1xx (100–199): Informational – Request received, processing ongoing.
  • 2xx (200–299): Success – Request was successfully processed.
  • 3xx (300–399): Redirection – Client must take additional action to complete the request.
  • 4xx (400–499): Client Error – Error due to the client’s request (e.g., missing data, unauthorized).
  • 5xx (500–599): Server Error – Error due to server-side issues.

3. Common Success Codes

200 OK – Standard success response. Used when a GET request succeeds.
201 Created – Resource successfully created (POST request).
202 Accepted – Request accepted but not yet processed.
204 No Content – Request succeeded but no content to return.

4. Common Client Error Codes

400 Bad Request – Invalid request or missing required fields.
401 Unauthorized – Client not authenticated.
403 Forbidden – Client authenticated but lacks permission.
404 Not Found – Requested resource or URL does not exist.
405 Method Not Allowed – HTTP method not supported for this endpoint.

5. Common Server Error Codes

500 Internal Server Error – Generic server error.
501 Not Implemented – Requested method not implemented.
503 Service Unavailable – Server temporarily unavailable.

6. Practical Examples

Example 1: Express & Postman GET Request
- Fetching users: returns 200 OK if user exists.
- Fetching non-existing user: returns 404 Not Found.
Example 2: Creating a New User
- POST request with complete data: returns 201 Created.
- POST request with missing fields: returns 400 Bad Request and a message "All fields are required".
Example 3: Authorization Errors
- Unauthenticated request: 401 Unauthorized.
- Authenticated but lacking permission: 403 Forbidden.
Example 4: Server Errors
- Logical or runtime error in server code: 500 Internal Server Error.
- Not implemented method: 501 Not Implemented.
- Temporary service unavailable: 503 Service Unavailable.

7. Redirection (3xx)

  • Used when the client should take additional action, e.g., URL shorteners or redirecting after payment.
  • Example: 301 (Moved Permanently), 302 (Found / Temporary Redirect).

8. Node.js & Express Implementation Tips

  • Set status code using res.status(code) before sending JSON: res.status(201).json(data).
  • Use nodemon to auto-restart server on file changes to avoid manual restarts during development.
  • Check all required fields in POST requests and respond with 400 if any field is missing.
